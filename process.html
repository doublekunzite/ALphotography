Development

Process Notes

A look behind the curtain at the problem-solving workflow used to build this site.

Key Iterations

Geometric Grid Stability Layout

Challenge: Layout shift was jarring during the "Quilt" to "Grid" transition.

"Make the movement smaller... keep the same grid dimensions between the two transitions."

Solution: Switched from Flexbox to CSS Grid with strict 5-column definition. Used negative/positive margins to create the overlap effect without reflowing the layout.

Mobile Drift Interaction UX

Challenge: Mobile users couldn't trigger the mouse-parallax effect, leading to static images.

"Achieve a similar effect on mobile... continuous, gentle floating animation."

Solution: Implemented a dual approach: Touchmove listeners for direct interaction + Sine-wave ambient animation for continuous drift.

Landscape Gallery Pivot Strategy

Challenge: Horizontal scroll interfered with vertical page scroll, breaking UX.

"The horizontal scroll still doesn't work... scrap this functionality and do something simpler."

Solution: Pivoted to a Vertical Slider. This preserved the full-screen immersion without fighting browser behavior.

Dynamic Aspect Ratios Logic

Challenge: "Polaroid" cards distorted when images were landscape vs. portrait.

"Can the 'polaroid' be wider for actual landscape dimensions...?"

Solution: Used JS onload to detect natural width/height, applying specific CSS classes to adjust container width dynamically.

Privacy & Security Hardening Security

Challenge: External dependencies and potential link vulnerabilities.

"I'd rather not use Google Fonts... are there any other possible security risks?"

Solution: Removed external font requests (System Fonts). Added rel="noopener noreferrer" to target="_blank" links to prevent tabnabbing. Audited innerHTML for XSS risks.

Workflow Strategies

Context Anchoring

Instead of trusting the AI to remember previous states, I provided the full updated codebase before requesting changes. This prevented "reversions" where the AI accidentally re-introduced old bugs or styling.

Exploratory Prompting

Rather than asking for an immediate solution ("Fix this"), I asked for options ("What are some possible display options?"). This allowed me to evaluate trade-offs between different UI patterns before committing code.

Critical Refinement

I requested the AI critique its own suggestions ("Any other ideas?") and explicitly asked to showcase problem-solving in the portfolio, turning the development process itself into a showcase artifact.
